Namespace sap.ui.test.qunit

The sap.ui.test.qunit namespace contains helper functionality for QUnit tests.

sap.ui.test.qunit.delayTestStart(iDelay?)
Delays the start of the test until everything is rendered or - if given - for the specified milliseconds. This function must be called before the first test function.
Parameters:
{int} iDelay?, Default: optional delay in milliseconds
sap.ui.test.qunit.triggerCharacterInput(oInput, sChar)
Programmtically triggers a keypress event on a specified input field target and appends the character to the value of this input field.
Parameters:
{string|DOMElement} oInput The ID of a DOM input field or a DOM input field which serves as target
{string} sChar Only the first char of the string will be passed via keypress event
See:
sap.ui.test.qunit.triggerKeypress
sap.ui.test.qunit.triggerEvent(sEventName, oTarget, oParams?)
Programmtically triggers an event specified by its name on a specified target with some optional parameters.
Parameters:
{string} sEventName The name of the browser event (like "click")
{string|DOMElement} oTarget The ID of a DOM element or a DOM element which serves as target of the event
{object} oParams?, Default: The parameters which should be attached to the event in JSON notation (depending on the event type).
See:
http://api.jquery.com/trigger/

sap.ui.test.qunit.triggerTouchEvent(sEventName, oTarget, oParams?)
Programmatically triggers a touch event specified by its name. The onEVENTNAME functions are called directly on the "nearest" control / element of the given target.
Parameters:
{string} sEventName The name of the touch event (touchstart, touchmove, touchend)
{string|DOMElement} oTarget The ID of a DOM element or a DOM element which serves as target of the event
{object} oParams?, Default: The parameters which should be attached to the event in JSON notation (depending on the event type).
