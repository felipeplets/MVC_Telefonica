File Structure

Assumptions:

  • Used UI5 core/libs could be anywhere (in a local resource folder or 'somewhere' else) ⇒ no resource folder is used
  • No reuse of the app in other apps ⇒ no need to model namespaces in the views/controller paths

The suggested application structure looks like this:

  • <webapp_root_folder>
    • Application.js
    • css … all CSS files go here (and potentially related images)
      • style.css … and potentially more stylesheets
    • ext … all reused third party java script goes here
    • i18n … all property files for localization go here
      • i18n.properties
      • i18n_de.properties
      • i18n_en.properties
      • …
    • model
    • util .. all helper classes go here
    • view .. all view and view controller files go here; all file names shall start with an upper case letter
    • img … all image files go here (png, jpg, gif, …)
    • <someNamespace> all on-the-fly controls defined within the application go here
      • <controlName>.js
    • index.html

Depending on the concrete application, not all of these resources may be required.

"someNamespace" could be like salesorders or it could also be a nested namespace (using folders): "sap/hcm/hiring". This namespace will be reflected in the package names of the controls: var btn = new salesorders.MyButton(); or new sap.hcm.hiring.CustomButton();

Such namespaces could also be used for Views/Controllers and helper classes, but this depends on the application. The "localResources" or "registerModulePath" statements (see the index.html section) need to be adapted accordingly.
