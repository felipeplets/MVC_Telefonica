Overview

Theme Designer is an application which creates custom themes for SAPUI5 applications based on a set of predefined theme parameters.

Prerequisites

Any SAPUI5 Application based on the core release 1.6.x or higher can be used within the Theme Designer. Certain features are only available with applications based on 1.8.x or later. This includes, but is not limited to, parameter scopes (such as quick or expert parameters).

The Theme Designer and the application which should be loaded within must both be available on the same domain (Same-Origin-Policy).

Features

By specifying a URL to your SAPUI5 application, the Theme Designer will load a list of available theme parameters given the set of included SAPUI5 libraries. Changing the parameters will update the application in real-time.

All parameter changes can be saved to the browser's local storage and will be available for a reload.

Custom CSS can be added andwill be reflected in the application preview in real-time.

The resulting theme along with any custom CSS can also be saved to the user's file system as a ZIP file.

Details

Loading an application

Specify the URL to your application on the start page of the Theme Designer and press the load button. The application will attempt to load all included theme parameters according to the SAPUI5 libraries available within the application (loaded via bootstrap configuration).

Changing parameter values

All parameters can be reached via the sections "Quick" or "Expert" depending on their definition. Changes to the value of a parameter will cause an update of the application preview in real-time upon submitting the value.

All color values can be manipulated using the Color Picker. Simply press the color preview to open this tool and manipulate the color value in either RGB or HSV color space.

Viewing changes

All changes made to theme parameters are available in the section "Change Log".

Resetting changes

Any change can be reset by selecting this entry in the "Change Log" section and clicking the "Reset" button in the toolbar above.

Custom CSS

Specify custom CSS which should be integrated into the application preview. The preview will be refreshed as soon as the changes are done. The custom CSS is applied after any other CSS elements specified in the HTML documents HEAD section. This data can thus be used to override any particular CSS rule.

Saving (local storage)

Changes to the current application preview due to parameter changes and custom CSS can be saved in the local browsers persistent storage (localStorage). Please note that this data might be erased by forcing a deletion of the browser's cache.

Saving (ZIP)

The resulting theme along with the custom CSS specified within Theme Designer can be downloaded as a ZIP file. This feature is currently not supported for Internet Explorer. Please note that the downloaded file must be renamed to a .zip extension when using Mozilla Firefox.

ZIP file contents

The contents of the created ZIP file correspond to the file structure for theme libraries created by SAP. You can simply extract these files and make them available on a web server of your choice. In order for your application to consume this theme, please check the corresponding documentation. In addition to the theme library files, any custom CSS will also be included in the ZIP container.
