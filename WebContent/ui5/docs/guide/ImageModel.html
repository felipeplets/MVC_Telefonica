Image Model

When referencing images in an application it is most convenient to collect all the access paths in one central place instead of pasting them all over the code. This can be achieved by storing them in a JSON file, making this available with a global JSON model - named "img" - and binding the controls against this model.

Storing image access paths in a JSON file: 

{
    "icon" : {
        "ui5" : "img/SAPUI5Icon.png",
        "cart" : "img/cart_TabButtonItem.png"
    }
}

Making images available with a global JSON model: 

ApplicationBase.extend("Application", {
    init : function() {
        var imgModel = new sap.ui.model.json.JSONModel("model/img.json");
        sap.ui.getCore().setModel(imgModel, "img");
        ...

Binding the controls against img model: 

this.page = new sap.m.Page({
    title : "Shopping Cart",
    icon : "{img>/icon/ui5}",
    ...

This approach allows even switching all images to a different set at runtime with one line of code (e.g. to provide differently colored image sets for different SAPUI5 themes).
