Implementing a Custom Model

To create a custom model implementation, you have to perform several steps:

  1. Extend the Model class and specify which binding modes the model should support (for example, two-way, one-way, one-time).
  2. Extend the Binding class to suit your specific binding or reuse the existing specific binding implementations PropertyBinding, ListBinding, and/or TreeBinding.
  3. To enable filtering functionality, use the Filter class with FilterOperator enum in your binding implementation.
  4. To enable sorting functionality, use the Sorter class in your binding implementation.

All the necessary classes can be found in the sap.ui.model namespace.

For more details, see the  data binding API.

As a starting point, take a look at the JSONModel implementation, which can be found in sap.ui.model.json.JSONModel.
