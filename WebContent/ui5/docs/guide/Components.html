Component Concept

UI5 components are independent and reusable parts of a UI5 Application. Hence, they may be developed by different development teams in different projects. An application can even use components from locations different from where this application itself is running. (Note: constraints due to cross-origin issues still apply, though.) Another advantage of componentization is not only the potential for distributed development, but also the encapsulation of closely related parts of an application into a particular component. This can help making the structure of an application and its code easier to understand and maintain.

Physically, a UI5 component is a folder whose name defines the name of the component. This folder contains all resources (like css files, internationalization files, views, images etc.) that are used within the particular component, as well as two additional files, Component.js and component.json. Note that all paths within a component are relative paths to the Component.js and not to e.g. an index.html file as could be the case in a regular web application.

  • Component.js represents the component controller and provides runtime metadata.
  • component.json contains metadata for design time, e.g. by the App Designer.

A component can be a faceless component or a UIComponent. Faceless components do not have any UI and could e.g. contain services providing data.

Component

Component.js contains the sap.ui.core.Component base class for faceless components as well as UI components. It provides the metadata for every component. The name parameter that gets passed into the component constructor represents the package name under which your component will be found.

UI Component

If a component needs to provide any representation on the screen, a UI component should be used. UIComponent.js extends Component.js and adds rendering functionality to the component.

Component Container

The ComponentContainer is a UI5 control that wraps a UIComponent. It can be injected into the UI5 control tree like any other regular control wherever it is needed.
