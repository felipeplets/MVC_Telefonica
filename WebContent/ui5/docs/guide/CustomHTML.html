Custom HTML

UI5 is open UI technology that allows you to easily incorporate custom HTML. This section of the best practice list the available options and provides guidance when to use what.


Options

HTML in XML Views

Within XML views you can easily incorporate XHTML which are converted 1:1 to HTML.

Pro/Con

  • Only for static HTML without databinding

HTML in HTML Views

Naturally an HTML view allows you to include custom HTML.

Pro/Con

  • Only for static HTML without databinding
  • If you want to use custom HTML inside a view this view type feels like a natural choice

Notepad Controls

With notepad controls you have a very good way to create reusable controls in a fast and structured way. This also provides you with the chance to use UI5 controls inside of your custom coding. We would recommended this way to bring in custom HTML, since it is good readable code and errors in the coding hardly have an impact on existing controls.

Sometimes it makes sense to inherit from an existing UI5 control. This is also possible with notepad controls and you can extend or change the existing coding. When you inherit controls you have to take care how to handle the inherited renderer and you really should know what you are doing. If you overwrite methods without calling the base method the control might not work anymore. Therefore check the existing methods and if you need to extend them, make sure you call the base method, too.

Pro/Con

  • Code encapsulated and reusable
  • Rerendering can be optimized
  • Works in XML and HTML views
  • Some more effort

HTML Control

An easy way to show simple content, is adding an HTML control to your app. Either write directly HTML or if you are using a model and databinding integrate it between the HTML by using a formatter. Remember that all changes to the model will rerender the control. This way of writing HTML makes sense, if you want to present some simple things, which aren't changed during runtime and you do not want to reuse it later.

Pro/Con

  • Code not encapsulated and reusable
  • Always rerenders completely on data changes
  • Does not work in XML and HTML views
  • Less effort

Loosing Themeability

The UI5 controls and their CSS are based on 'theme parameters' which allow to use the 'Theme Designer' to very easily brand an application with a customer specific color scheme. Todays web standards do not support CSS parameter replacement at runtime. As such the theme parameters have to be replaced with the real values during the build process of the UI5 controls.

This implies that at the current stage the usage of theme parameters is not available for custom html. Custom HTML which is build with the above options cannot be themed in the Theme Designer.
